Angular Material

What is Angular Material?

Angular Material is a prebuilt framework for angular components made by the angular team. The components are made to be responsive and to run well on mobil and desktop environments. Material itself was designed by google to synthesis design principles as well as class platform unified look and feel.

Principles

1: Material is a metaphor

Material design handles elements as if they are 3 dimensions, just like material in the real world. This means using things like shadow to give this illusion.

2: Bold,graphic, and Intentional

Color pallets and typography is chosen intentionally to create an environment. 3: Motion provides meaning

Animations in material are used to convey what is happening on the page and in the application. this is used rather than jarring motions.

Material Icons

Material icons can be configured with a global svg icon registry, or it can be configure to apply icons provided directly from google.

Material Table

Material tables are configures to automatically complete themselves using a pre-defined data source as described below. Each row in the table is defined in a ng-container that includes a matColumnDef that corresponds to an element in the display columns array defined in the backend. Each row must have a valid entry in this array.

Material Filter


Angular Material also allows you to add sorting functionality to your table, allowing users to only look at the relevant data to them.

Material Sort


Sorting in Angular material in a mat table only requires matSort in the table and mat-sort-column on any column that you would like to be able to sort by. In the back end you also need to include the @ViewChild(MatSort) sort: MatSort;

Dialogs and Popups


The MatDialog module allows you to define a material stylized component which can be opened and closed with the option of choosing how to handle different inputs. Dialogs can have specified material components such as mat-dialog-title, content, and actions. A button can be created to interact with the dialog by including [mat-dialog-close]="true/false". Dialogs are created on the fly, so the component factory needs a description on how to create it.

Theming

Themes in Angular Material include a set of icons ans well as sets of color for prebuilt theming of your website. A style is required for all functions of angular material to work, though you can provide a custom theming file which imports the mat-core() sass mixin which includes all common styles and defines a theme data structure that composes multiple palettes and is built from either mat-light-theme or mat-dark-theme.

Ex:

@import '~@angular/material/theming';
@include mat-core();

$candy-app-primary: mat-palette($mat-indigo);
$candy-app-accent: mat-palette($mat-pink, A200, A100, A400);

$candy-app-warn: mat-palette($mat-red);

/ $candy-app=theme: mat-light-theme(
$candy-app-primary,
$candy-app-accent,
$candy-app-warn);

@include angular-material-theme($candy-app-theme);

As you can see, material theming is designed on sass files. These custom themes can be defined in our style.sass or in another file which we import in our style.css file. Themes can also be defined and reset with events in our application, allowing for different themes to be applied dynamically or even when routing.
